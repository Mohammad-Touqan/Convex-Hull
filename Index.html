🔃 CLO-2: SORTING & SEARCHING — How each algorithm finds the hull
📐 GRAHAM SCAN — USES SORTING
Step 1 — Find anchor (O(n)):
Scan all points for the lowest-y point. This is the starting point — guaranteed to be on the hull.
Step 2 — Polar angle SORT (O(n log n)):
Sort all other points by angle from the anchor going counterclockwise. Uses cross products instead of slow trigonometry.
⚠ This sort is ESSENTIAL — without it the algorithm produces wrong results!
Step 3 — Stack sweep (O(n)):
Process sorted points one by one. Because they're sorted, each cross product check is final — no need to revisit. Total sweep is O(n).
📊 Polar angles of your current points (sorted left → right = CCW order):
■ Anchor (pivot)   ■ Sorted CCW order — each bar = one point's angle
❌ What if we SKIP the sort?
The stack would process points in random order. It can't tell if a turn is locally or globally valid — it would keep wrong points and miss correct ones, producing a self-intersecting polygon instead of a convex hull.
🎁 JARVIS MARCH — USES SEARCHING
No sorting needed!
Instead of sorting upfront, Jarvis does a linear search for the best next point at every single step.
Each round — O(n) linear search:
From the current hull point, scan ALL n points and find the one most counterclockwise. This is the search step — it runs h times total (once per hull vertex).
Total cost = O(n × h):
n = total points scanned per round
h = number of hull vertices found
When h is small (like 3), this is only 3n — much faster than Graham's n log n!

🧠 CLO-3: DESIGN PARADIGM — How these algorithms are designed
🎁 JARVIS MARCH = GREEDY ALGORITHM
What is Greedy?
A greedy algorithm makes the best choice available right now at each step, without looking ahead or going back. It never reconsiders a decision.
Jarvis greedy choice:
"From where I am now, which point is the most counterclockwise?" — pick it, add it to the hull, never go back.
🔄 GREEDY STEPS — WATCH DURING JARVIS:
1 Start at leftmost point (anchor) — guaranteed on hull
2 Scan ALL points — find most CCW (the greedy scan)
3 Commit! Add that point to hull — no going back ✓
4 Repeat from new point until back at start
Why greedy is CORRECT here:
The most-CCW point from any hull vertex is always the correct next hull vertex. No future information can make a different choice better. This is called the greedy choice property.
⚡ GRAHAM SCAN = SORT + SWEEP PARADIGM
What is Sort + Sweep?
Spend time upfront organizing data (sort), then solve the problem in one fast pass (sweep). The sort makes each sweep decision O(1) instead of O(n).
📋 STACK INVARIANT — WATCH DURING GRAHAM:
1 Sort by polar angle — one-time O(n log n) investment
2 For each point: cross product test → LEFT = push ✓
3 RIGHT turn = POP — stack always stays convex
4 Stack at end = complete convex hull ✓
❌ Why NOT Divide & Conquer or DP?
D&C (merge hull): splits points, solves halves, merges — works but more complex code and higher constant factor.
DP: no overlapping subproblems exist here — each hull decision is independent — so DP doesn't apply.
Wrong greedy example: picking the nearest point each time gives a wrong answer — it misses far-out hull vertices entirely.
Convex Hull
Smallest convex polygon enclosing all input points. Like a rubber band stretched around all points and released.
Cross Product
cross(O,A,B)=(A-O)×(B-O). Positive=left/CCW. Negative=right/CW. Zero=collinear. No trig needed.
Output Sensitive
Jarvis is output-sensitive: O(nh). For 3 hull pts among 1000 → 3000 ops. Graham always O(n log n).
Anchor Point
Both start from lowest-leftmost point — guaranteed on hull. Graham sorts from it; Jarvis wraps from it.
Stack Invariant
Graham's stack always holds a valid convex chain. Pop right/collinear turns to restore convexity before pushing.
Polar Angle Sort
Sorting by atan2(dy,dx) from pivot gives CCW order. Enables single-pass O(n) sweep with O(1) stack ops.
